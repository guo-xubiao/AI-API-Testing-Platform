# 当前任务卡片

## 当前任务

Stage 1 - Task 5：实现用户注册与登录模块

## 任务目标

实现后续订单流程所需的用户注册与登录基础能力，并保持统一响应和异常处理规范。

## 已完成内容

- 已完成 Stage 1 - Task 1：创建 Spring Boot 项目骨架。
- 已完成 Stage 1 - Task 2：设计统一响应与异常体系。
- 已完成 Stage 1 - Task 3：设计订单系统领域模型。
- 已完成 Stage 1 - Task 4：实现商品模块。
- 已具备 `User` 实体、`UserRepository`、统一响应对象和全局异常处理器。

## 本任务要求

- 为 `User` 补充登录所需的密码字段，密码不得通过接口响应返回。
- 创建 `UserController`、`UserService`、注册和登录请求 DTO。
- 实现 `POST /api/users/register`：校验必填项与用户名唯一性，完成用户注册。
- 实现 `POST /api/users/login`：校验用户名与密码，返回不含密码的用户信息或登录结果。
- 注册重复用户名、用户不存在或密码错误时，使用现有业务异常体系返回明确错误。
- 接口统一使用 `ApiResponse`，并对请求参数使用校验注解。
- 不实现订单创建、支付、取消、发货、完成、退款等后续流程。

## 验收标准

- 注册与登录接口均可用，且返回统一响应格式。
- 任何响应 DTO 或实体序列化都不得泄露密码。
- 重复注册、用户不存在、密码错误和非法参数均能得到可预期的错误响应。
- 不提前实现订单、支付、退款业务流程。
- 能在可访问 Maven 依赖仓库的环境中完成 `mvn test` 或等价应用启动验证。
