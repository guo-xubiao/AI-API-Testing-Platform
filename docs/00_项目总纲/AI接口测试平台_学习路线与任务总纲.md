# AI 接口测试平台：学习路线与任务总纲

## 项目目标

通过任务驱动方式，逐步完成一个 AI 驱动的接口测试平台。

最终平台应具备项目管理、环境管理、接口管理、单接口调试、测试用例管理、场景编排、变量提取、断言配置、执行日志、测试报告、定时任务、通知提醒、OpenAPI 导入、AI 生成测试用例、AI 生成场景、AI 分析失败原因、登录权限和 Dashboard 等能力。

## 推进原则

- 一次只做一个任务。
- 当前任务未通过，不进入下一任务。
- 每个任务必须有明确验收标准。
- 任务通过后更新进度日志，并发布下一任务。
- 不提前实现后续阶段功能。

## 阶段规划

1. Stage 1：OrderFlow Demo 被测业务系统
2. Stage 2：接口测试平台基础版
3. Stage 3：测试用例管理
4. Stage 4：场景编排
5. Stage 5：变量提取与断言系统
6. Stage 6：执行记录与测试报告
7. Stage 7：AI 生成测试用例
8. Stage 8：AI 生成场景编排
9. Stage 9：AI 分析失败原因
10. Stage 10：产品化增强

## 当前路线

当前从 Stage 1 开始，先搭建 `orderflow-demo`，让后续接口测试平台有一套真实可测的订单履约接口。
